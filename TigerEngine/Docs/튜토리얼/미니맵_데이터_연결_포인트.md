# 미니맵_데이터_연결_포인트

---

## 목적
미니맵 UI 연동을 위해 필요한 데이터/이벤트 연결 포인트를 정리합니다.
(좌표, 진행도, 보유 여부)

## 데이터 연결 포인트

### 1) 플레이어 좌표 → 미니맵 핑 위치
- 제공 함수: `MiniMapTestScript::GetPlayerMapPos()`
- 사용처: `MiniMapManager::OnUpdate()`
- 대상 UI 오브젝트:
  - Base: `UI_MiniMap_Base` (RectTransform 필요)
  - Ping: `UI_MiniMap_MainPing` (RectTransform 필요)
- 계산 방식:
  - `WorldToMiniMap(worldPos, worldMin, worldMax, mapSize)`
  - `pingPos = basePos + local` (Pivot/Anchor 변경 시 오프셋 검증 필요)

### 2) 지도 조각 진행도
- 제공 함수: `MiniMapTestScript::GetProgress01()`
- 의미: `collectedPieces / totalPieces` (0~1)
- 연결 대상:
  - 게이지 UI: `UI_MiniMap_GaugeFill` (RectTransform width 조절 방식)
- 참고:
  - 현재 게이지는 Fill 셰이더 대신 RectTransform의 `Size.x`를 진행도에 맞게 줄이는 방식

### 3) 재료(ingredient) 보유 여부 / 개수 표시
- 보유/개수는 인벤토리에서 관리
- UI는 아이콘 활성/비활성로 표시하는 것을 권장
- 데이터 연결 포인트:
  - 인벤토리 획득 이벤트에서 미니맵/아이콘 UI로 전달

### 4) 지도 조각 붙이기(임시)
- UI 조각 오브젝트:
  - `UI_MiniMap_Piece_01` ~ `UI_MiniMap_Piece_06`
- 활성화 트리거:
  - `MiniMapManager::TriggerPieceActive(index, true)`
- 획득 순서와 무관하게 index 매핑으로 표시 가능

## 이벤트 트리거 (획득 시점 Push 방식)
미니맵에서 수집하지 않고, 인벤토리/상호작용 코드에서 Push 호출합니다.

- 지도 조각 획득(표시): `MiniMapManager::TriggerPieceActive(index, true)`
- 지도 조각 카운트 갱신(진행도): `MiniMapTestScript::TriggerPieceCollected()`
- 재료 획득(표시): `MiniMapManager::TriggerItemPing(index, worldPos)`
- 재료 카운트 갱신: `MiniMapTestScript::TriggerItemCollected()`

> 연결 권장 위치: `PlayerController::SerachObjectInteraction()` 내부에서 `inventory->AddItem()` 호출 직후
